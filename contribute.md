# Contribute to CSLibre

CSLibre thrives because of people like you — educators, students, developers, and learners who believe in free, accessible education. Every contribution, no matter how small, helps thousands of students learn better.

## Ways to Contribute

### 1. Report Errors & Issues

Found a mistake? Spot a typo? Notice outdated information?

**How to report:**

- Visit our [GitHub Issues](https://github.com/CSLibre/CSLibre.github.io/issues) page
- Click "New Issue"
- Describe the error clearly with page URL and details
- We'll review and fix it promptly

**What to report:**

- Factual errors in content
- Broken code examples
- Typos and grammar mistakes
- Broken links or missing resources
- Confusing explanations

### 2. Improve Existing Content

Help make our content clearer, more accurate, and more helpful.

**What you can do:**

- Add missing explanations or examples
- Improve clarity of existing explanations
- Add diagrams or visual aids
- Include more practice questions
- Share better code examples
- Add real-world applications

**How to contribute:**

1. Fork the [CSLibre repository](https://github.com/CSLibre/CSLibre.github.io)
2. Make your changes in your fork
3. Submit a Pull Request with clear description
4. We'll review and merge approved changes

### 3. Create New Content

Share your teaching expertise by creating new lessons, examples, or resources.

**Content we need:**

- Topic explanations and notes
- Solved programming examples
- Practice questions with solutions
- Project ideas with documentation
- Exam preparation materials
- Video tutorials or presentations
- Infographics and visual learning aids

**Content guidelines:**

- Must align with ICSE, ISC, or CBSE curriculum
- Should be clear and student-friendly
- Include working code examples (if applicable)
- Provide step-by-step explanations
- Follow our [content standards](/about/#our-content-standards)

**How to submit:**

1. Check if the topic already exists
2. Draft your content in Markdown format
3. Include proper front matter (see examples)
4. Submit via Pull Request with description
5. Email us at [computer.science.libre@gmail.com](mailto:computer.science.libre+newcontent@gmail.com) if you need help

### 4. Translate Content

Help make CSLibre accessible to more students by translating content into regional languages.

**Languages needed:**

- English
- Hindi


**Translation guidelines:**

- Keep technical terms in English where appropriate
- Maintain clarity and simplicity
- Preserve code examples as-is
- Follow proper language grammar and style=

### 5. Review & Test Content

Help ensure quality by reviewing and testing our content.

**What reviewers do:**

- Test all code examples to ensure they work
- Check explanations for accuracy
- Verify exam alignment and pattern matching
- Suggest improvements for clarity
- Test content on different devices

### 6. Share Your Projects

Inspire others by sharing your student projects built using CSLibre resources.

**What to share:**

- Project description and objectives
- Complete working code
- Setup and usage instructions
- Screenshots or demo videos
- Challenges faced and solutions

**Submit projects** via Pull Request or email [computer.science.libre@gmail.com](mailto:computer.science.libre+projects@gmail.com)

### 7. Spread the Word

Help more students discover CSLibre.

**How to help:**

- Share CSLibre with your classmates and friends
- Tell your teachers and school administrators
- Post about us on social media
- Write blog posts or reviews
- Create video tutorials using our content (with attribution)
- Add CSLibre to educational resource lists

### 8. Provide Feedback & Suggestions

Your feedback helps us improve and grow.

**Share your thoughts on:**

- What topics you'd like to see added
- How we can improve existing content
- New features or tools you'd like
- Your learning experience with CSLibre
- What's working well and what's not

**Send feedback** to [computer.science.libre@gmail.com](mailto:computer.science.libre+feedback@gmail.com)

### 9. Technical Contributions

Help improve the platform itself.

**Technical help needed:**

- Website performance optimization
- Accessibility improvements
- Bug fixes and feature additions
- Interactive coding playground development
- Mobile app development
- Build tools and automation

**For developers:**

Check our [GitHub repository](https://github.com/CSLibre/CSLibre.github.io/issues) for open issues tagged with `good-first-issue` or `help-wanted`.

## Contribution Guidelines

### Quality Standards

All contributions must meet these standards:

**Accuracy** — Content must be factually correct and technically sound.

**Clarity** — Explanations should be clear and easy to understand for students.

**Completeness** — Include all necessary information, examples, and explanations.

**Originality** — Content should be original or properly attributed if adapted.

**Curriculum Alignment** — Must align with ICSE, ISC, or CBSE syllabi.

**Code Quality** — All code must be tested, working, and well-commented.

### Attribution

When contributing content adapted from other sources:

- Clearly mention the original source
- Ensure the source allows redistribution
- Follow their attribution requirements
- Add your own explanation and context

### Code of Conduct

All contributors must follow our code of conduct:

**Be respectful** — Treat everyone with respect and kindness.

**Be constructive** — Provide helpful, constructive feedback.

**Be inclusive** — Welcome people of all backgrounds and skill levels.

**Be patient** — Remember that everyone is learning.

**Give credit** — Acknowledge others' work and contributions.

**No harassment** — Zero tolerance for harassment or discrimination.

## Recognition

We value every contribution and recognize our contributors:

**All contributors** are listed in our [Contributors](/contributors/) page.

**Major contributors** receive special recognition and acknowledgment.

**Top contributors** are featured on our homepage.

**Regular contributors** may be invited to join the core team.

## Getting Started

Ready to contribute? Here's how to begin:

### Step 1: Choose Your Contribution Type

Pick what interests you most from the list above.

### Step 2: Check Existing Content

- Browse the website to see what exists
- Check [open issues](https://github.com/CSLibre/CSLibre.github.io/issues) on GitHub
- Avoid duplicating existing work

### Step 3: Plan Your Contribution

- Outline what you want to create or improve
- Gather necessary resources and references
- Check curriculum alignment if applicable

### Step 4: Create & Submit

- Follow our formatting guidelines
- Test everything thoroughly
- Submit via GitHub Pull Request or email

### Step 5: Respond to Feedback

- Be open to suggestions and improvements
- Make requested changes promptly
- Ask questions if anything is unclear

## Frequently Asked Questions

**Do I need to know GitHub to contribute?**

No! You can email your contributions directly to us, and we'll add them for you.

**Will I get credit for my contributions?**

Yes! All contributors are acknowledged on our Contributors page and in commit history.

**Can students contribute?**

Absolutely! Student contributions are especially valuable as you understand what helps learners best.

**Is there a minimum contribution size?**

No! Even fixing a single typo is a valuable contribution.

**How long does review take?**

Most contributions are reviewed within 3-5 days. Larger contributions may take longer.

**Can I contribute anonymously?**

Yes, but we encourage you to take credit for your valuable work!

**What if my contribution is rejected?**

We'll always explain why and suggest improvements. Most contributions are accepted with minor edits.
